# 飓风信号网站建设方案

## 一、项目概述

### 1.1 项目定位
- **网站名称**：飓风信号
- **品牌名称**：飓风信号
- **核心理念**：从"孤狼"到"联合舰队"的转变，打造未来私募基金的"预备队"和"造船厂"
- **目标用户**：有交易基础或潜力的交易员，寻求团队协作和职业发展的专业人士

### 1.2 设计风格
- **配色方案**：基于 `ui.html` 中已设计的配色
  - 主色调：`#00C9FF`（青蓝色）
  - 辅助色：`#00FFAA`（青绿色）
  - 强调色：`#6C63FF`（紫色）
  - 背景色：`#F7FAFC`（浅灰）
- **设计风格**：现代、专业、科技感，体现交易平台的严谨性和创新性

## 三、核心页面详细规划

### 3.1 首页 (/)

#### 3.1.1 Hero区域（品牌展示）
- **视觉元素**：
  - 大标题："从孤狼到联合舰队"
  - 副标题："飓风信号 - 未来私募基金的预备队"
  - 背景：可考虑视频背景或动态粒子效果
  - CTA按钮：引导至"实战训练营"和"投资第一课"
- **设计要点**：强品牌感，第一眼传达核心理念

#### 3.1.2 品牌故事概览
- **内容**：精简版品牌故事（3-4段）
- **视觉**：图文结合，可加入创始人照片或团队照片
- **CTA**：链接到"关于我们"页面查看完整故事

#### 3.1.3 职业发展路径
- **展示方式**：时间轴或阶梯式展示
- **路径内容**：
  ```
  交易员 → 监理 → 团队领导 → 合伙人
  ```
- **设计**：每个阶段展示职责、要求、收益预期

#### 3.1.4 团队收益展示
- **数据可视化**：
  - 平均收益率
  - 团队规模
  - 成功案例
  - 用户满意度
- **展示方式**：数据卡片 + 图表

#### 3.1.5 CTA引导区
- **主要引导**：
  - "申请实战训练营"
  - "了解投资第一课"
  - "免费心理测评"

### 3.3 实战训练营 (/training)

#### 3.3.1 训练营介绍
- **核心价值点**：

  - 真实交易环境

  - 第一阶段：基础培训

- **申请流程**：
  1. 投递简历至指定邮箱
  2. 初步筛选
  3. 面试评估
  4. 进入训练营
- **筛选标准**：
  - 学历要求
  - 经历要求
  - 其他要求

### 3.6 测评 (/assessment)

#### 3.6.1 测评介绍
- **内容**：
  - 测评目的：评估交易员的心理素质和交易能力
  - 测评内容：20道题目，涵盖5个维度
    - 风险承受能力（4题）
    - 情绪控制能力（4题）
    - 决策能力（4题）
    - 纪律性（4题）
    - 压力管理（4题）
  - 测评时长：约10-15分钟
  - 结果说明：提供详细的维度分析和建议

#### 3.6.2 测试题目
- **题目来源**：`心理评测.md` 文件
- **题型**：单选题，每题4个选项
- **评分机制**：每个选项对应1-4分，分数越高表示该维度能力越强
- **交互功能**：
  - 答题进度显示
  - 可保存进度（本地存储）
  - 题目随机顺序（可选）
  - 返回修改答案

#### 3.6.3 结果分析
- **展示内容**：
  - 各维度得分（雷达图展示）
  - 总体评分
  - 能力分析（每个维度的详细分析）
  - 改进建议和指导
  - 适合的交易风格建议
- **数据存储**：结果可保存到本地，或提交到服务器（可选）

## 四、技术实现建议

### 4.1 技术栈
- **框架**：Next.js（已初始化）
- **样式**：CSS Modules + CSS Variables（集中管理配色）
- **国际化**：next-intl 或 next-i18next（**优先级：先中文，英文后续开发**）
- **UI组件**：可考虑使用 shadcn/ui 或自定义组件
- **状态管理**：React Context 或 Zustand（如需要）
- **表单处理**：React Hook Form
- **数据获取**：Next.js API Routes 或外部API
- **内容管理**：**JSON配置文件（网页静态数据）+ Markdown（博客内容）**

### 4.2 设计系统
- **配色方案集中管理**：
  - 位置：`styles/theme.css` 或 `styles/variables.css`
  - 使用CSS Variables，所有颜色定义在一个文件中
  - 支持主题切换（如需要）
- **基于**：`ui.html` 中的配色方案
- **组件库**：建立可复用的组件库
  - Button组件
  - Card组件
  - Form组件
  - Navigation组件
  - 等

### 4.3 内容管理系统（已确认）

**采用方案：JSON配置文件 + Markdown**

#### 4.3.1 JSON配置文件（管理网页静态数据）
- **位置**：`content/` 目录
- **结构**：
  ```
  content/
  ├── zh/                    # 中文内容（优先开发）
  │   ├── home.json
  │   ├── about.json
  │   ├── training.json
  │   ├── faq.json
  │   └── ...
  └── en/                    # 英文内容（后续开发）
      ├── home.json
      ├── about.json
      ├── training.json
      ├── faq.json
      └── ...
  ```
- **优点**：简单、版本控制友好、无需数据库
- **适用**：页面文案内容、FAQ、统计数据、配置信息等静态数据

#### 4.3.2 Markdown文件（管理博客内容）
- **位置**：`content/blog/` 目录
- **结构**：
  ```
  content/blog/
  ├── zh/                    # 中文博客
  │   ├── 2024/
  │   │   ├── 2024-01-15-文章标题.md
  │   │   └── ...
  │   └── ...
  └── en/                    # 英文博客（后续开发）
      ├── 2024/
      │   └── ...
      └── ...
  ```
- **格式**：Markdown文件，使用Front Matter存储元数据
  ```markdown
  ---
  title: 文章标题
  date: 2024-01-15
  author: 作者名
  category: 交易策略
  tags: [交易, 策略, 分析]
  excerpt: 文章摘要
  ---
  
  文章正文内容...
  ```
- **优点**：易于编写、支持富文本、SEO友好、版本控制友好
- **适用**：博客文章、教程、资讯等动态内容

### 4.4 国际化（i18n）方案

**优先级：先开发中文版本，英文版本后续开发**

#### 4.4.1 第一阶段：中文版本
- **路由**：直接使用根路径（`/`, `/about`, `/training` 等）
- **内容文件**：仅使用 `content/zh/` 目录下的文件
- **文案文件**：仅使用 `messages/zh/` 目录下的文件

#### 4.4.2 第二阶段：英文版本（后续开发）
- **技术选型**：next-intl（推荐）或 next-i18next
- **文件结构**：
  ```
  messages/
  ├── zh/                    # 中文文案（已开发）
  │   ├── common.json
  │   ├── home.json
  │   └── ...
  └── en/                    # 英文文案（后续开发）
      ├── common.json
      ├── home.json
      └── ...
  ```
- **路由策略**：路径前缀（推荐）
  - 中文：`/zh/`, `/zh/about`, `/zh/training`（或保持根路径）
  - 英文：`/en/`, `/en/about`, `/en/training`
- **语言切换**：在导航栏提供语言切换按钮

### 4.5 SEO优化
- **页面元数据**：每个页面设置合适的 title、description
- **结构化数据**：使用 JSON-LD
- **sitemap**：生成网站地图
- **robots.txt**：配置爬虫规则

### 4.6 性能优化
- **图片优化**：使用 Next.js Image 组件
- **代码分割**：按需加载
- **缓存策略**：合理设置缓存

## 七、设计规范

### 7.1 颜色系统（集中管理）

**文件位置**：`styles/theme.css` 或 `styles/variables.css`

```css
:root {
  /* 主色调 */
  --primary: #00C9FF;
  --primary-light: #E6F9FF;
  --primary-dark: #0099CC;
  
  /* 辅助色 */
  --secondary: #00FFAA;
  --secondary-light: #E6FFF5;
  --secondary-dark: #00CC88;
  
  /* 强调色 */
  --accent: #6C63FF;
  --accent-light: #E6E5FF;
  --accent-dark: #4A42CC;
  
  /* 文字颜色 */
  --text-dark: #2D3748;
  --text-light: #718096;
  --text-muted: #A0AEC0;
  
  /* 背景颜色 */
  --bg-white: #FFFFFF;
  --bg-light: #F7FAFC;
  --bg-gray: #EDF2F7;
  
  /* 边框颜色 */
  --border: #E2E8F0;
  --border-light: #F1F5F9;
  
  /* 状态颜色 */
  --success: #48BB78;
  --success-light: #C6F6D5;
  --warning: #ED8936;
  --warning-light: #FEEBC8;
  --danger: #F56565;
  --danger-light: #FED7D7;
  
  /* 阴影 */
  --shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.05);
  --shadow-md: 0 4px 6px rgba(0, 0, 0, 0.1);
  --shadow-lg: 0 10px 15px rgba(0, 0, 0, 0.1);
  --shadow-xl: 0 20px 25px rgba(0, 0, 0, 0.1);
  
  /* 渐变 */
  --gradient-primary: linear-gradient(to right, var(--primary), var(--secondary));
  --gradient-accent: linear-gradient(to right, var(--primary), var(--accent));
}
```

**使用方式**：
- 在全局CSS文件中引入：`@import './styles/theme.css'`
- 在组件中使用：`color: var(--primary)`
- 修改配色：只需修改 `theme.css` 文件即可全局生效

### 7.2 字体规范
- **主字体**：'Segoe UI', 'PingFang SC', 'Microsoft YaHei', sans-serif
- **标题字体**：可考虑使用更醒目的字体（如无衬线粗体）

### 7.3 间距系统
- **基础单位**：8px
- **常用间距**：8px, 16px, 24px, 32px, 48px, 64px
