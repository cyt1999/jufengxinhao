# 内容管理说明

## 概述

本网站采用**集中式内容管理**方案，所有文案内容和配色方案都集中在特定文件中，方便后续更新和维护。

---

## 一、配色方案管理

### 文件位置
`styles/theme.css`

### 使用方法
1. **修改配色**：直接编辑 `styles/theme.css` 文件中的 CSS 变量
2. **全局生效**：修改后，所有使用该变量的地方都会自动更新
3. **无需重启**：修改 CSS 变量后，刷新页面即可看到效果

### 示例
```css
/* 修改主色调 */
--primary: #00C9FF;  /* 改为其他颜色，如 #FF6B6B */
```

### 在组件中使用
```tsx
// 在组件中使用配色变量
<div style={{ color: 'var(--primary)' }}>
  这是主色调文字
</div>

// 或在 CSS 文件中使用
.my-class {
  color: var(--primary);
  background: var(--bg-light);
}
```

## 二、内容管理

**采用方案：JSON配置文件（静态数据）+ Markdown（博客内容）**

### 2.1 JSON配置文件（管理网页静态数据）

#### 文件结构
```
content/
├── zh/                    # 中文内容（优先开发）
│   ├── home.json
│   ├── about.json
│   ├── training.json
│   ├── faq.json
│   ├── assessment/
│   │   └── questions.json  # 测评题目
│   └── ...
└── en/                    # 英文内容（后续开发）
    ├── home.json
    ├── about.json
    ├── training.json
    ├── faq.json
    └── ...
```

#### 使用方法

**1. 更新页面内容**
直接编辑对应的 JSON 文件即可，例如：
- 修改首页内容：编辑 `content/zh/home.json`
- 修改关于我们：编辑 `content/zh/about.json`
- 修改测评题目：编辑 `content/zh/assessment/questions.json`

**2. 添加新内容**
在对应的 JSON 文件中添加新的字段即可，例如：
```json
{
  "hero": {
    "title": "新标题",
    "description": "新描述"
  }
}
```

**3. 在代码中使用**
```tsx
// 加载内容
import homeContent from '@/content/zh/home.json';

// 使用内容
<h1>{homeContent.hero.title}</h1>
<p>{homeContent.hero.description}</p>
```

### 2.2 Markdown文件（管理博客内容）

#### 文件结构
```
content/blog/
├── zh/                    # 中文博客（优先开发）
│   ├── 2024/
│   │   ├── 2024-01-15-如何制定有效的交易策略.md
│   │   ├── 2024-01-10-风险管理的重要性.md
│   │   └── ...
│   ├── 2025/
│   │   └── ...
│   └── ...
└── en/                    # 英文博客（后续开发）
    ├── 2024/
    │   └── ...
    └── ...
```

#### 文件命名规范

**格式**：`YYYY-MM-DD-文章标题.md`

**规则**：
- 日期格式：`YYYY-MM-DD`（年-月-日）
- 文章标题：使用中文，简短清晰，便于识别
- 文件名中的标题会用于生成 URL slug（自动去除日期前缀）
- 示例：`2024-01-15-如何制定有效的交易策略.md` → URL: `/blog/如何制定有效的交易策略`

**注意事项**：
- 文件名中的标题不要包含特殊字符（如 `/`、`\`、`:`、`*`、`?`、`"`、`<`、`>`、`|`）
- 建议使用中文标题，简洁明了
- 日期必须准确，用于文章排序和显示

#### Front Matter 元数据格式

每篇博客文章必须在文件开头包含 Front Matter（用 `---` 包裹的 YAML 格式元数据）：

```markdown
---
title: 文章标题
date: 2024-01-15
author: 作者名
category: 交易策略
tags: [交易, 策略, 分析]
excerpt: 文章摘要，会显示在文章列表页，建议控制在100-200字
---

文章正文内容...
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `title` | 字符串 | 是 | 文章标题 | `如何制定有效的交易策略` |
| `date` | 字符串 | 是 | 发布日期（YYYY-MM-DD） | `2024-01-15` |
| `author` | 字符串 | 否 | 作者名称，默认为"飓风信号团队" | `飓风信号团队` |
| `category` | 字符串 | 是 | 文章分类，用于分类筛选 | `交易策略`、`风险管理`、`心理分析` |
| `tags` | 数组 | 否 | 文章标签，用于搜索和关联 | `[交易, 策略, 分析]` |
| `excerpt` | 字符串 | 是 | 文章摘要，显示在列表页 | `交易策略是成功交易的基础...` |

**分类建议**：
- `交易策略` - 交易策略相关文章
- `风险管理` - 风险管理相关文章
- `心理分析` - 交易心理相关文章
- `市场分析` - 市场分析相关文章
- `技术分析` - 技术分析相关文章
- `新手入门` - 新手入门相关文章

#### Markdown 语法支持

文章正文支持标准 Markdown 语法：

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体文字**
*斜体文字*

- 无序列表项1
- 无序列表项2

1. 有序列表项1
2. 有序列表项2

[链接文字](https://example.com)

![图片描述](图片路径)

> 引用文字

`行内代码`

```代码块
function example() {
  return 'Hello';
}
```

| 表格列1 | 表格列2 |
|---------|---------|
| 内容1   | 内容2   |
```

#### 使用方法

**1. 创建新文章**

步骤：
1. 确定文章发布日期和标题
2. 在 `content/blog/zh/` 目录下找到对应年份的文件夹（如 `2024/`），如果没有则创建
3. 创建 Markdown 文件，文件名格式：`YYYY-MM-DD-文章标题.md`
4. 在文件开头添加 Front Matter 元数据
5. 编写文章正文内容
6. 保存文件

示例：
```bash
# 创建文件
content/blog/zh/2024/2024-02-01-交易员的一天.md

# 文件内容
---
title: 交易员的一天
date: 2024-02-01
author: 飓风信号团队
category: 新手入门
tags: [交易员, 日常, 工作]
excerpt: 了解职业交易员的日常工作流程，从早盘准备到收盘复盘，全面了解交易员的一天。
---

# 交易员的一天

作为一名职业交易员，每天的工作都有严格的流程...
```

**2. 更新现有文章**

- 直接编辑对应的 Markdown 文件
- 可以修改 Front Matter 元数据（如分类、标签、摘要）
- 可以修改文章正文内容
- 保存后刷新网站即可看到更新

**3. 删除文章**

- 直接删除对应的 Markdown 文件
- 文章会自动从网站中移除

**4. 文章在网站上的显示**

- **列表页**：`/blog` - 显示所有文章，支持分类筛选和搜索
- **详情页**：`/blog/文章标题` - 显示完整文章内容
- **自动排序**：按发布日期倒序排列（最新的在前）
- **自动分类**：根据 Front Matter 中的 `category` 字段自动分类
- **搜索功能**：支持按标题、摘要、标签搜索

#### 注意事项

1. **文件编码**：确保文件使用 UTF-8 编码，避免中文乱码
2. **Front Matter 格式**：必须使用 YAML 格式，前后用 `---` 包裹
3. **日期格式**：必须使用 `YYYY-MM-DD` 格式，如 `2024-01-15`
4. **分类一致性**：建议使用统一的分类名称，便于管理
5. **摘要长度**：建议控制在 100-200 字，过长会被截断
6. **图片路径**：如果文章包含图片，建议将图片放在 `public/images/blog/` 目录下
7. **特殊字符**：文件名中避免使用特殊字符，使用中文或英文
8. **备份**：修改前建议先备份文件

#### 技术实现说明

**在代码中的使用**（开发者参考）：

```tsx
// 获取所有文章
import { getAllPosts } from '@/lib/blog';
const posts = getAllPosts();

// 获取单篇文章
import { getPostBySlug } from '@/lib/blog';
const post = await getPostBySlug('如何制定有效的交易策略');

// 按分类筛选
import { getPostsByCategory } from '@/lib/blog';
const posts = getPostsByCategory('交易策略');

// 搜索文章
import { searchPosts } from '@/lib/blog';
const posts = searchPosts('交易策略');
```

**文件读取机制**：
- 系统会自动递归读取 `content/blog/zh/` 目录下的所有 `.md` 文件
- 自动解析 Front Matter 元数据
- 自动将 Markdown 内容转换为 HTML
- 自动按日期排序

## 三、国际化文案管理

**优先级：先开发中文版本，英文版本后续开发**

### 文件结构
```
messages/
├── zh/                    # 中文文案（优先开发）
│   ├── common.json        # 通用文案（按钮、导航等）
│   ├── home.json
│   ├── about.json
│   └── ...
└── en/                    # 英文文案（后续开发）
    ├── common.json
    ├── home.json
    ├── about.json
    └── ...
```

### 使用方法

#### 1. 更新通用文案
编辑 `messages/zh/common.json`（中文版本优先）

#### 2. 在代码中使用
```tsx
// 使用国际化（第一阶段仅使用中文）
import { useTranslations } from 'next-intl';

function MyComponent() {
  const t = useTranslations('common');
  
  return (
    <button>{t('buttons.submit')}</button>
  );
}
```

#### 3. 后续添加英文版本
- 复制 `messages/zh/` 目录结构到 `messages/en/`
- 翻译所有文案内容
- 配置国际化路由（路径前缀或子域名）

## 四、更新流程

### 更新配色方案
1. 打开 `styles/theme.css`
2. 修改对应的 CSS 变量
3. 保存文件
4. 刷新页面查看效果

### 更新页面内容
1. 打开对应的 JSON 文件（如 `content/zh/home.json`）
2. 修改对应的字段值
3. 保存文件
4. 刷新页面查看效果

### 添加新文章（博客）
1. 在 `content/blog/zh/` 目录下找到对应年份的文件夹（如 `2024/`），如果没有则创建
2. 创建 Markdown 文件，文件名格式：`YYYY-MM-DD-文章标题.md`
3. 在文件开头添加 Front Matter 元数据（必须包含 title、date、category、excerpt）
4. 编写文章正文内容（支持 Markdown 语法）
5. 保存文件，刷新网站即可看到新文章

**详细步骤请参考"2.2 Markdown文件（管理博客内容）"章节**

### 添加新语言（后续开发）
1. 在 `content/` 目录下创建新的语言文件夹（如 `content/en/`）
2. 复制现有语言的文件结构
3. 翻译所有内容
4. 在代码中配置新语言

## 六、最佳实践

1. **命名规范**：使用有意义的字段名，如 `hero.title` 而不是 `t1`
2. **结构清晰**：保持 JSON 结构清晰，便于理解和维护
3. **注释说明**：在 JSON 文件中添加注释（虽然 JSON 不支持注释，但可以在字段名中体现）
4. **一致性**：保持中英文内容结构一致，便于维护

## 八、重要说明

### 8.1 支付方式
- **不接入支付接口**：网站不接入在线支付功能
- **线下支付**：采用线下支付方式（微信收款码、支付宝收款码、银行账户等）
- **支付流程**：用户完成支付后，通过表单提交支付凭证，人工审核确认

### 8.2 国际化优先级
- **第一阶段**：仅开发中文版本
- **第二阶段**：后续开发英文版本
- **代码结构**：虽然先开发中文，但代码结构需考虑国际化扩展

### 8.3 内容管理方式
- **静态数据**：使用 JSON 配置文件管理（页面文案、FAQ、统计数据等）
- **博客内容**：使用 Markdown 文件管理（支持 Front Matter 元数据）
