# 内容管理说明

## 概述

本网站采用**集中式内容管理**方案，所有文案内容和配色方案都集中在特定文件中，方便后续更新和维护。

---

## 一、配色方案管理

### 文件位置
`styles/theme.css`

### 使用方法
1. **修改配色**：直接编辑 `styles/theme.css` 文件中的 CSS 变量
2. **全局生效**：修改后，所有使用该变量的地方都会自动更新
3. **无需重启**：修改 CSS 变量后，刷新页面即可看到效果

### 示例
```css
/* 修改主色调 */
--primary: #00C9FF;  /* 改为其他颜色，如 #FF6B6B */
```

### 在组件中使用
```tsx
// 在组件中使用配色变量
<div style={{ color: 'var(--primary)' }}>
  这是主色调文字
</div>

// 或在 CSS 文件中使用
.my-class {
  color: var(--primary);
  background: var(--bg-light);
}
```

## 二、内容管理

**采用方案：JSON配置文件（静态数据）+ Markdown（博客内容）**

### 2.1 JSON配置文件（管理网页静态数据）

#### 文件结构
```
content/
├── zh/                    # 中文内容（优先开发）
│   ├── home.json
│   ├── about.json
│   ├── training.json
│   ├── faq.json
│   ├── assessment/
│   │   └── questions.json  # 测评题目
│   └── ...
└── en/                    # 英文内容（后续开发）
    ├── home.json
    ├── about.json
    ├── training.json
    ├── faq.json
    └── ...
```

#### 使用方法

**1. 更新页面内容**
直接编辑对应的 JSON 文件即可，例如：
- 修改首页内容：编辑 `content/zh/home.json`
- 修改关于我们：编辑 `content/zh/about.json`
- 修改测评题目：编辑 `content/zh/assessment/questions.json`

**2. 添加新内容**
在对应的 JSON 文件中添加新的字段即可，例如：
```json
{
  "hero": {
    "title": "新标题",
    "description": "新描述"
  }
}
```

**3. 在代码中使用**
```tsx
// 加载内容
import homeContent from '@/content/zh/home.json';

// 使用内容
<h1>{homeContent.hero.title}</h1>
<p>{homeContent.hero.description}</p>
```

### 2.2 Markdown文件（管理博客内容）

#### 文件结构
```
content/blog/
├── zh/                    # 中文博客（优先开发）
│   ├── 2024/
│   │   ├── 2024-01-15-文章标题.md
│   │   └── ...
│   └── ...
└── en/                    # 英文博客（后续开发）
    ├── 2024/
    │   └── ...
    └── ...
```

#### 文件格式
使用 Front Matter 存储元数据：
```markdown
---
title: 文章标题
date: 2024-01-15
author: 作者名
category: 交易策略
tags: [交易, 策略, 分析]
excerpt: 文章摘要
---

文章正文内容...

支持 Markdown 格式：
- 列表
- **粗体**
- *斜体*
- [链接](url)
- 代码块
```

#### 使用方法

**1. 创建新文章**
- 在 `content/blog/zh/` 目录下按年份创建文件夹
- 创建 Markdown 文件，文件名格式：`YYYY-MM-DD-文章标题.md`
- 添加 Front Matter 元数据
- 编写文章内容

**2. 更新文章**
- 直接编辑对应的 Markdown 文件
- 修改 Front Matter 或正文内容

**3. 在代码中使用**
```tsx
// 加载 Markdown 文件
import fs from 'fs';
import matter from 'gray-matter';

const fileContents = fs.readFileSync('content/blog/zh/2024/2024-01-15-文章标题.md', 'utf8');
const { data, content } = matter(fileContents);

// data 包含 Front Matter 元数据
// content 包含文章正文
```

## 三、国际化文案管理

**优先级：先开发中文版本，英文版本后续开发**

### 文件结构
```
messages/
├── zh/                    # 中文文案（优先开发）
│   ├── common.json        # 通用文案（按钮、导航等）
│   ├── home.json
│   ├── about.json
│   └── ...
└── en/                    # 英文文案（后续开发）
    ├── common.json
    ├── home.json
    ├── about.json
    └── ...
```

### 使用方法

#### 1. 更新通用文案
编辑 `messages/zh/common.json`（中文版本优先）

#### 2. 在代码中使用
```tsx
// 使用国际化（第一阶段仅使用中文）
import { useTranslations } from 'next-intl';

function MyComponent() {
  const t = useTranslations('common');
  
  return (
    <button>{t('buttons.submit')}</button>
  );
}
```

#### 3. 后续添加英文版本
- 复制 `messages/zh/` 目录结构到 `messages/en/`
- 翻译所有文案内容
- 配置国际化路由（路径前缀或子域名）

## 四、更新流程

### 更新配色方案
1. 打开 `styles/theme.css`
2. 修改对应的 CSS 变量
3. 保存文件
4. 刷新页面查看效果

### 更新页面内容
1. 打开对应的 JSON 文件（如 `content/zh/home.json`）
2. 修改对应的字段值
3. 保存文件
4. 刷新页面查看效果

### 添加新文章（博客）
1. 在 `content/blog/zh/` 目录下按年份创建文件夹（如 `2024/`）
2. 创建 Markdown 文件，文件名格式：`YYYY-MM-DD-文章标题.md`
3. 添加 Front Matter 元数据
4. 编写文章内容

### 添加新语言（后续开发）
1. 在 `content/` 目录下创建新的语言文件夹（如 `content/en/`）
2. 复制现有语言的文件结构
3. 翻译所有内容
4. 在代码中配置新语言

## 五、注意事项

1. **JSON 格式**：确保 JSON 文件格式正确，可以使用 JSON 验证工具检查
2. **编码**：所有文件使用 UTF-8 编码
3. **备份**：修改前建议先备份文件
4. **版本控制**：所有内容文件都纳入 Git 版本控制，方便追踪变更

---

## 六、最佳实践

1. **命名规范**：使用有意义的字段名，如 `hero.title` 而不是 `t1`
2. **结构清晰**：保持 JSON 结构清晰，便于理解和维护
3. **注释说明**：在 JSON 文件中添加注释（虽然 JSON 不支持注释，但可以在字段名中体现）
4. **一致性**：保持中英文内容结构一致，便于维护

## 八、重要说明

### 8.1 支付方式
- **不接入支付接口**：网站不接入在线支付功能
- **线下支付**：采用线下支付方式（微信收款码、支付宝收款码、银行账户等）
- **支付流程**：用户完成支付后，通过表单提交支付凭证，人工审核确认

### 8.2 国际化优先级
- **第一阶段**：仅开发中文版本
- **第二阶段**：后续开发英文版本
- **代码结构**：虽然先开发中文，但代码结构需考虑国际化扩展

### 8.3 内容管理方式
- **静态数据**：使用 JSON 配置文件管理（页面文案、FAQ、统计数据等）
- **博客内容**：使用 Markdown 文件管理（支持 Front Matter 元数据）
