# 联系表单配置说明

## 当前实现

已创建 Next.js API 路由 (`/api/contact`) 来处理表单提交，使用 Resend 服务发送邮件。

## 配置步骤

### 1. 安装依赖

使用 pnpm 安装 Resend：

```bash
pnpm add resend
```

### 2. 配置环境变量

在项目根目录创建 `.env.local` 文件（参考 `.env.local.example`）：

```env
# Resend API Key（从 https://resend.com 获取）
RESEND_API_KEY=re_5MVU3CrZ_A1AsAw69wyaBNiuv3QdnS5Ly

# 发件人邮箱
# 使用 onboarding@resend.dev 无需验证，但仅用于测试
# 生产环境建议使用已验证的域名邮箱
RESEND_FROM_EMAIL=onboarding@resend.dev

# 接收联系表单的邮箱地址
CONTACT_EMAIL=jufengxinhao@gmail.com
```

### 3. 验证配置

配置完成后，重启开发服务器：

```bash
pnpm dev
```

## 使用说明

- 表单提交后，邮件会发送到 `CONTACT_EMAIL` 配置的邮箱
- 发件人使用 `RESEND_FROM_EMAIL` 配置的邮箱
- 回复邮件会直接发送到用户填写的邮箱地址

## 生产环境建议

1. **验证域名**：
   - 在 Resend 中验证您的域名
   - 使用您自己的域名邮箱作为发件人（如：contact@jufengxinhao.com）

2. **更新环境变量**：
   ```env
   RESEND_FROM_EMAIL=contact@jufengxinhao.com
   ```

## 注意事项

- `.env.local` 文件已添加到 `.gitignore`，不会被提交到代码仓库
- 生产环境部署时，需要在部署平台配置相同的环境变量
- Resend 免费额度：每月 3000 封邮件
